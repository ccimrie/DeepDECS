DeepDECS

Discrete-Event Controller Synthesis for Autonomous Systems with
Deep-Learning Perception Components

Radu Calinescu, Calum Imrie, Ravi Mangal, Genaína Nunes Rodrigues, Corina Păsăreanu, Misael Alpizar Santana, and Gricel Vázquez

Github repo

Many autonomous systems use deep neural networks (DNNs) to perceive the world around them. However, no DNN is perfect, which can be devastating in safety-critical applications. DeepDECS is a controller synthesis method that captures the uncertainty of DNN perception components through the use of DNN verification techniques. From this, it produces correct-by-construction controllers for such autonomous systems.

The diagram on the right depicts the DeepDECS controller synthesis process, including its four inputs. The first is a discrete-time Markov Chain (DTMC) modelling the behaviour of the autonomous system within its application (assuming perfect perception); the second is a set of requirements specified in probabilistic computation tree logic (PCTL). The third input is a pre-trained perception DNN that will be utilised in the autonomous system, and the fourth is a representative test dataset for this DNN. These last two inputs are utilised, along with \(n\) verification methods, in Stage 1 ('DNN uncertainty quantification') of DeepDECS to derive a set of \(2^{n}\) confusion matrices. These confusion matrices are then used to estimate the DNN classification/misclassification probabilities for an augmented, DNN-perception DTMC obtained automatically in Stage 2 ('Model augmentation') of DeepDECS. The augmented model contains an accurate representation of the imperfect DNN perception component. Finally, Stage 3 ('Controller synthesis') of DeepDECS employs probabilisitc model checking techniques to generate the autonomous system controller synthesis.

Augment Tool

Incorporating uncertainty

An augmentation tool was created to generate the parametric DTMC (pDTMC) to synthesise the pareto optimal controllers. The tool requires a perfect perception pDTMC and the confusion matrices generated from step 1 of DeepDECS. DeepDECS Tutorial

How fast to drive on potential ice?

Say you have an autonomous car driving down a road and the car is equipped with a perception unit (i.e. a DNN). The aim of the perception unit is to determine if the next segment of road is icy or not. Having this information is invaluable as the car should not drive fast on icy roads, but the perception unit is not perfect so how do we best ascertain a safety critical controller? Case Studies

Mobile robot collision avoidance & Shared autonomy for driving

To evaluate DeepDECS two case studies were studied. The first is a robot in a warehouse where it must travel from one point to another in a known map. The robot navigates by traversing waypoints, and finds the shortest route using Dijkstra's algorithm. The robot, however, is not the only agent operating in the warehouse, and now the robot must also minimise dangerous collisions. At each waypoint the robot first detects if a collider is present, and if so uses a DNN to predict whether a collision will occur if the robot proceeds. The video on the left shows the robot utilising a safe controller synthesised from the DeepDECS framework.

The second case study focused on shared control in autonomous driving, see the image on the left. Given the complexity there will be times while driving that the autonomous car will not be able to operate, thus will need to transfer control to the human driver. This therefore requires the human driver to be alert at all times should a transfer of control be required. The car predicts the human driver's attentiveness using a DNN, and needs to decide which alerts to use if the human driver is not paying attention, such as lights, vibration and sound. For this case study real data was used of human drivers in a simulated scenario.

For more detail on the case studies you can read the paper. Access to the code and models can be found on this repo.
